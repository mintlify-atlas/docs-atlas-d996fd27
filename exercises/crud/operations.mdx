## Overview

This guide walks through implementing all four CRUD operations using the MVC (Model-View-Controller) pattern. Each operation follows the same architectural flow:

```
View (Form/UI) → Controller (Logic) → Model (ORM) → Database
```

## Create Operation

Create new user accounts with form validation and password hashing.

### View: Registration Form

The view presents an HTML form for user input:

```html registroUsuario.view.php
<form action="./scripts/CRUD/registroUsuario.controller.php" method="POST">
    <fieldset>
        <legend>Registro de Usuario</legend>
        <label for="cedula">Cédula de identidad:</label>
        <input type="text" id="cedula" name="cedula" placeholder="cedula" required>
        <label for="nombres">Nombres:</label>
        <input type="text" id="nombres" name="nombres" placeholder="nombres" required>
        <label for="apellidos">Apellidos:</label>
        <input type="text" id="apellidos" name="apellidos" placeholder="apellidos" required>
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" placeholder="email" required>
        <label for="user_pass">Contraseña:</label>
        <input type="password" id="user_pass" name="user_pass" placeholder="contraseña" required>
        <button type="submit">Registrar</button>
    </fieldset>
</form>
```

<Note>
The form uses HTML5 validation with `required` attributes and `type="email"` for basic client-side validation.
</Note>

### Controller: Process Registration

The controller handles the POST request and interacts with the model:

```php registroUsuario.controller.php
<?php
require_once  "database.php";
require_once  "orm.class.php";
require_once  "Usuario.class.php";

$database = new Database();
$conexion = $database->getConexion();

$usuarioModel = new Usuario($conexion);

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $data = [
        'cedula' => $_POST['cedula'],
        'nombres' => $_POST['nombres'],
        'apellidos' => $_POST['apellidos'],
        'email' => $_POST['email'],
        'user_pass' => password_hash($_POST['user_pass'], PASSWORD_BCRYPT),
    ];

    $usuarioModel->save($data);

    header('Location: ../../index.php#crud-basico-con-php');
    exit();
}
```

#### Controller Flow

<Steps>
  <Step title="Initialize dependencies">
    Load database, ORM, and Usuario classes
  </Step>
  
  <Step title="Establish database connection">
    Create Database instance and get PDO connection
  </Step>
  
  <Step title="Check request method">
    Verify the request is POST to process form submission
  </Step>
  
  <Step title="Build data array">
    Extract POST data and hash the password using bcrypt
  </Step>
  
  <Step title="Save to database">
    Call the model's `save()` method to insert the record
  </Step>
  
  <Step title="Redirect">
    Redirect to prevent form resubmission (Post-Redirect-Get pattern)
  </Step>
</Steps>

### Security: Password Hashing

The controller uses PHP's built-in password hashing:

```php
'user_pass' => password_hash($_POST['user_pass'], PASSWORD_BCRYPT)
```

<ParamField path="password" type="string" required>
  The plain text password from user input
</ParamField>

<ParamField path="algorithm" type="constant" required>
  `PASSWORD_BCRYPT` - Uses bcrypt algorithm with automatic salt generation
</ParamField>

<Note>
**Why bcrypt?** The bcrypt algorithm is computationally expensive, making brute-force attacks impractical. Each hash includes a random salt automatically.
</Note>

## Read Operation

Retrieve and display all users from the database.

### Controller: Fetch Data

The controller queries the database and prepares data for the view:

```php listarUsuario.controller.php
<?php
require_once  "database.php";
require_once  "orm.class.php";
require_once  "Usuario.class.php";

$database = new Database();
$conexion = $database->getConexion();
$usuarioModel = new Usuario($conexion);
$usuarios = $usuarioModel->getAll();
```

<Note>
This controller is included directly by the view file rather than being accessed via HTTP request.
</Note>

### View: Display Users Table

The view renders the data in an HTML table:

```php listarUsuario.view.php
<?php
require_once "listarUsuario.controller.php";

if (count($usuarios) > 0) {
    echo "<table>
            <thead>
                <tr>
                    <th>ID</th>
                    <th>Cedula</th>
                    <th>Nombre</th>
                    <th>Apellido</th>
                    <th>Email</th>
                </tr>
            </thead>
            <tbody>";
    foreach ($usuarios as $usuario) {
        echo "<tr>
                <td>{$usuario['id']}</td>
                <td>{$usuario['cedula']}</td>
                <td>{$usuario['nombres']}</td>
                <td>{$usuario['apellidos']}</td>
                <td>{$usuario['email']}</td>
              </tr>";
    }
    echo "  </tbody>
        </table>";
} else {
    echo "<p>No hay usuarios registrados.</p>";
}
```

#### View Logic

<Steps>
  <Step title="Include controller">
    Load the controller to get the `$usuarios` array
  </Step>
  
  <Step title="Check for data">
    Use `count()` to determine if there are any users
  </Step>
  
  <Step title="Render table or message">
    Display table if users exist, otherwise show "no users" message
  </Step>
  
  <Step title="Loop through users">
    Use `foreach` to generate a table row for each user
  </Step>
</Steps>

<Note>
**Security Warning**: This example outputs user data directly without escaping. In production, use `htmlspecialchars()` to prevent XSS attacks:

```php
echo "<td>" . htmlspecialchars($usuario['nombres']) . "</td>";
```
</Note>

## Update Operation

Modify existing user records with the same security measures as creation.

### Controller: Process Update

```php modificarUsuario.controller.php
<?php
require_once  "database.php";
require_once  "orm.class.php";
require_once  "Usuario.class.php";

$database = new Database();
$conexion = $database->getConexion();

$usuarioModel = new Usuario($conexion);

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $data = [
        'cedula' => $_POST['cedula'],
        'nombres' => $_POST['nombres'],
        'apellidos' => $_POST['apellidos'],
        'email' => $_POST['email'],
        'user_pass' => password_hash($_POST['user_pass'], PASSWORD_BCRYPT),
    ];

    $usuarioModel->updateById($_POST['id'], $data);

    header('Location: ../../index.php#crud-basico-con-php');
    exit();
}
```

### Key Differences from Create

<CardGroup cols={2}>
  <Card title="Uses updateById()" icon="pen-to-square">
    Calls `updateById()` method instead of `save()`
  </Card>
  
  <Card title="Requires ID" icon="key">
    Must include the user's ID from `$_POST['id']` to identify the record
  </Card>
  
  <Card title="Same validation" icon="check">
    Uses identical password hashing and data structure as create
  </Card>
  
  <Card title="Empty value filtering" icon="filter">
    ORM automatically filters empty fields, updating only provided values
  </Card>
</CardGroup>

### Update Flow

```
1. User submits edit form with ID
2. Controller receives POST data including ID
3. Controller hashes new password (if provided)
4. Controller calls updateById(id, data)
5. ORM filters empty values
6. ORM builds dynamic UPDATE statement
7. Database updates only non-empty fields
8. Redirect to prevent resubmission
```

<Note>
The ORM's `updateById()` method filters empty values, so you can send the entire form data without worrying about overwriting fields the user didn't want to change.
</Note>

## Delete Operation

Remove users from the database with simple ID-based deletion.

### Controller: Process Deletion

```php borrarUsuario.controller.php
<?php
require_once  "database.php";
require_once  "orm.class.php";
require_once  "Usuario.class.php";

$database = new Database();
$conexion = $database->getConexion();
$usuarioModel = new Usuario($conexion);

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $id = $_POST['id'];
    $usuarioModel->deleteById($id);
}

header('Location: ../../index.php#crud-basico-con-php');
exit();
```

### Delete Flow

<Steps>
  <Step title="Initialize database and model">
    Set up database connection and Usuario model instance
  </Step>
  
  <Step title="Verify POST request">
    Check that the request method is POST for security
  </Step>
  
  <Step title="Extract ID">
    Get the user ID from POST data
  </Step>
  
  <Step title="Delete record">
    Call `deleteById()` to remove the user from database
  </Step>
  
  <Step title="Redirect">
    Always redirect after deletion to refresh the page
  </Step>
</Steps>

<Note>
**No confirmation**: This implementation deletes immediately without confirmation. In production, add JavaScript confirmation or a dedicated confirmation page.
</Note>

## Security Best Practices

### 1. Input Validation

Always validate user input before processing:

```php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // Validate required fields
    if (empty($_POST['email']) || empty($_POST['cedula'])) {
        die('Required fields missing');
    }
    
    // Validate email format
    if (!filter_var($_POST['email'], FILTER_VALIDATE_EMAIL)) {
        die('Invalid email format');
    }
    
    // Validate cedula length
    if (strlen($_POST['cedula']) !== 8) {
        die('Cedula must be 8 digits');
    }
    
    // Process the data...
}
```

### 2. Password Security

<CardGroup cols={2}>
  <Card title="Use password_hash()" icon="lock">
    Always hash passwords with `PASSWORD_BCRYPT` or `PASSWORD_ARGON2ID`
  </Card>
  
  <Card title="Verify with password_verify()" icon="key">
    Use `password_verify($input, $hash)` to check passwords during login
  </Card>
  
  <Card title="Never log passwords" icon="eye-slash">
    Never log, display, or transmit plain text passwords
  </Card>
  
  <Card title="Enforce strong passwords" icon="shield">
    Require minimum length, complexity, and common password checks
  </Card>
</CardGroup>

### 3. SQL Injection Prevention

The ORM uses prepared statements automatically:

```php
// SECURE: Uses prepared statements
$stmt = $this->db->prepare("SELECT * FROM usuarios WHERE id = :id");
$stmt->bindParam(':id', $id);
$stmt->execute();

// INSECURE: Never do this!
// $sql = "SELECT * FROM usuarios WHERE id = $id";
// $this->db->query($sql);
```

### 4. XSS Prevention

Escape output when displaying user data:

```php
// SECURE: Escaped output
echo "<td>" . htmlspecialchars($usuario['nombres'], ENT_QUOTES, 'UTF-8') . "</td>";

// INSECURE: Direct output
// echo "<td>{$usuario['nombres']}</td>";
```

### 5. CSRF Protection

Add CSRF tokens to forms:

```php
// Generate token
session_start();
if (empty($_SESSION['csrf_token'])) {
    $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
}

// In form
echo '<input type="hidden" name="csrf_token" value="' . $_SESSION['csrf_token'] . '">';

// Verify token
if ($_POST['csrf_token'] !== $_SESSION['csrf_token']) {
    die('CSRF token validation failed');
}
```

## Error Handling

Implement proper error handling in controllers:

```php
try {
    $database = new Database();
    $conexion = $database->getConexion();
    $usuarioModel = new Usuario($conexion);
    
    if ($_SERVER['REQUEST_METHOD'] === 'POST') {
        $data = [
            'cedula' => $_POST['cedula'],
            'nombres' => $_POST['nombres'],
            'apellidos' => $_POST['apellidos'],
            'email' => $_POST['email'],
            'user_pass' => password_hash($_POST['user_pass'], PASSWORD_BCRYPT),
        ];
        
        $result = $usuarioModel->save($data);
        
        if ($result) {
            header('Location: ../../index.php?success=1');
        } else {
            header('Location: ../../index.php?error=1');
        }
    }
} catch (PDOException $e) {
    // Log error (don't display to user in production)
    error_log('Database error: ' . $e->getMessage());
    die('An error occurred. Please try again later.');
} catch (Exception $e) {
    error_log('General error: ' . $e->getMessage());
    die('An unexpected error occurred.');
}
```

## Common Issues and Solutions

### Issue: Duplicate Entry Error

```
Error: Duplicate entry 'email@example.com' for key 'email'
```

**Solution**: Check for existing records before inserting:

```php
$existing = $usuarioModel->getByEmail($_POST['email']);
if ($existing) {
    die('Email already registered');
}
```

### Issue: Empty Password Hash

```
Warning: password_hash() expects parameter 1 to be string
```

**Solution**: Validate password exists:

```php
if (empty($_POST['user_pass'])) {
    die('Password required');
}
```

### Issue: Headers Already Sent

```
Warning: Cannot modify header information - headers already sent
```

**Solution**: Ensure no output before `header()` calls:

```php
// No echo, whitespace, or HTML before this:
header('Location: ../../index.php');
exit(); // Always call exit after redirect
```
