## Introduction

This collection of exercises teaches you how to build a complete **CRUD (Create, Read, Update, Delete)** application using PHP with a clean MVC (Model-View-Controller) architecture and a custom ORM (Object-Relational Mapping) pattern.

## What You'll Learn

Through these exercises, you'll master:

- Building database-driven applications with MySQL
- Implementing the MVC architectural pattern
- Creating a reusable ORM class for database operations
- Handling user authentication with password hashing
- Managing database connections with PDO
- Processing HTTP requests and form data

## The User Management System

The exercises implement a complete **user management system** with the following features:

- **User Registration**: Create new user accounts with validation
- **User Listing**: Display all registered users in a table
- **User Updates**: Modify existing user information
- **User Deletion**: Remove users from the database

### Technology Stack

- **Backend**: PHP 8.2+
- **Database**: MySQL/MariaDB
- **Architecture**: MVC Pattern
- **Data Layer**: Custom ORM Class
- **Security**: PDO Prepared Statements, Password Hashing (bcrypt)

## MVC Architecture

The application follows a clean separation of concerns:

```
┌─────────────┐
│    View     │  (HTML Forms & Tables)
│   .view.php │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Controller  │  (Business Logic)
│.controller  │
│    .php     │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│    Model    │  (ORM + Database)
│  .class.php │
└─────────────┘
```

## Project Structure

```
CRUD/
├── database.php                    # Database connection class
├── orm.class.php                   # Base ORM with CRUD methods
├── usuario.class.php               # User model extending ORM
├── registroUsuario.controller.php  # Create user
├── registroUsuario.view.php        # Registration form
├── listarUsuario.controller.php    # Read users
├── listarUsuario.view.php          # Users table
├── modificarUsuario.controller.php # Update user
├── modificarUsuario.view.php       # Edit form
├── borrarUsuario.controller.php    # Delete user
├── borrarUsuario.view.php          # Delete confirmation
```

## Database Schema

The system uses a single `usuarios` table with the following structure:

| Column     | Type         | Constraints              |
|------------|--------------|-------------------------|
| id         | INT(11)      | PRIMARY KEY, AUTO_INCREMENT |
| cedula     | VARCHAR(8)   | UNIQUE, NOT NULL        |
| nombres    | VARCHAR(255) | NOT NULL                |
| apellidos  | VARCHAR(255) | NOT NULL                |
| email      | VARCHAR(255) | UNIQUE, NOT NULL        |
| user_pass  | VARCHAR(18)  | UNIQUE, NOT NULL        |

<Note>
Passwords are hashed using PHP's `password_hash()` with bcrypt algorithm before storage.
</Note>

## Key Concepts

### PDO (PHP Data Objects)

The application uses PDO for database interactions, providing:
- Database abstraction layer
- Prepared statements to prevent SQL injection
- Exception-based error handling
- Support for different database drivers

### Object-Oriented PHP

The codebase demonstrates:
- Class inheritance (Usuario extends ORM)
- Encapsulation with protected properties
- Constructor dependency injection
- Method chaining and return values
